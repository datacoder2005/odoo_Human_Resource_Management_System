You are a senior full-stack developer and UI engineer.

Build a Human Resource Management System (HRMS) web app using:

Frontend:
- React + Vite
- React Router
- Context API or Redux Toolkit for auth/user state
- Tailwind CSS or clean modern CSS
- Responsive layout

Backend:
- Node.js + Express
- JWT authentication
- REST APIs
- bcrypt for password hashing
- role-based authorization middleware

Database:
- MongoDB Atlas
- Mongoose models

Project goal:
Create a clean, hackathon-ready HRMS prototype focused on the employee dashboard, profile page, and payroll page. The system must support role-based access for Employee and Admin/HR users. Employees should only access their own records, while Admin/HR can view and manage all employee data.

Core product requirements:
1. Authentication
- Sign Up and Sign In pages
- Sign up fields: employeeId, name, email, password, role
- Roles: Employee, Admin/HR
- Password hashing with bcrypt
- JWT login
- Protected routes
- Redirect user to the correct dashboard based on role
- Show errors for invalid login or duplicate email

2. Employee Dashboard
- This is the landing page after login
- Clean modern dashboard with:
  - top navigation bar
  - employee avatar/profile dropdown
  - welcome section
  - clickable cards
- Dashboard cards must be clickable and route to:
  - My Profile
  - Attendance
  - Leave Requests
  - Payroll
  - Logout
- Show quick summary widgets such as:
  - today’s attendance status
  - leave balance
  - latest salary status
- For Employee role, show only their own data
- For Admin/HR role, show management-focused cards and broader data

3. Profile Page
- Create a detailed employee profile page
- Show:
  - profile photo/avatar
  - name
  - employee ID
  - role
  - department
  - designation
  - email
  - phone
  - address
  - joining date
  - documents section
  - salary info preview if allowed
- Employee can edit only limited fields:
  - address
  - phone
  - profile picture
- Admin/HR can edit all employee details
- Include Edit Profile and Save/Cancel flow
- Design should feel like a professional HR system card layout

4. Payroll Page
- Create a dedicated payroll/salary page
- Show salary details in a clean table or card layout:
  - basic salary
  - allowances
  - deductions
  - bonus
  - net salary
  - pay period/month
  - payment status
- Employee view must be read-only
- Admin/HR can view all employees’ salary data and edit salary structure
- Add a salary breakdown section
- Add a payroll summary header at the top
- Add a back button to dashboard/profile

5. Role-based Access
- Employees can only see and edit their own profile information
- Employees can only see their own attendance/payroll
- Admin/HR can view all employees
- Admin/HR can switch between employee records
- Hide or disable unauthorized actions on the frontend
- Enforce permissions on backend too, not only in the UI

6. Backend APIs
Create REST endpoints such as:
- POST /api/auth/register
- POST /api/auth/login
- GET /api/users/me

- PUT /api/users/me
- PUT /api/users/:id (admin only)
- GET /api/payroll/me

- GET /api/attendance/me
- GET /api/attendance/:userId (admin only)
